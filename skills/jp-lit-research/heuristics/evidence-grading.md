# 典拠評価（evidence_grading）

## 候補確度（書誌・関連性）

このラベルは主に「書誌・候補としての確からしさ」を表す。文献の内容把握や学術的位置づけの確からしさは、`確認:` と `本文:` で別に示す。

- 書誌の確からしさ: その文献が存在し、題名・著者・発行年等が合っているか
- 関連性の確からしさ: 要旨・目次・本文・引用経由などで主題との関係を確認できたか
- 内容把握の確からしさ: エージェントが本文・要旨・目次・書評等のどこまで確認したか
- 調査上の確認優先度: 次に確認すべき順番。文献価値の確定ではない

本文未読でも仮整理してよいが、`候補確度`, `根拠`, `確認`, `本文`, `次` を明示する。

### 書誌確認済み（Bibliographically confirmed）

以下のいずれかが揃っている:

- 書誌情報が複数の信頼できる DB で一致
- DOI / CRID / NDL書誌ID などの永続 ID が確認済み

`書誌確認済み` は本文読了を意味しない。OCR ヒット、ページ画像確認、原文ページ確認は `根拠:`、`確認:`、`本文:` のレイヤーへ書き、書誌確度の条件に混ぜない。本文または該当箇所を読んだ場合だけ `確認: 本文` / `本文: 確認済み` とする。

### 有力候補（Probable）

- 書誌情報は取得できたが現物未確認
- OCR 全文でヒットしたが目視確認なし（`jp_lit_search_fulltext` のみ）
- タイトル・著者・年が一致するが、同名別資料の可能性が排除できない
- レファ協の調査事例に挙がっているが一次情報で確認していない

### 弱い候補（Weak）

- タイトルや著者が類似しているがすべての書誌事項が一致しない
- LLM の推測による関連語でヒットしたが確認していない
- OCR のみで誤認識の疑いが高い（文字化け・断片）
- Web 上の二次情報のみ

## 内容把握と二次情報

根拠は、何についての根拠かを分ける。

| 根拠 | 主に使えること | 注意 |
|------|----------------|------|
| 本文 | 内容把握、主張、論証、引用候補 | 該当箇所を読んだ場合だけ使う |
| 要旨 | 内容把握の仮整理 | 要旨の範囲に限定する |
| 目次 | 扱う論点・構成・章立て | 本文結論の根拠にはしない |
| DB 件名 | 主題推定・検索語展開 | 件名だけで内容を断定しない |
| 学術書評・専門誌書評 | 受容・評価・論争状況 | 書評本文を読んだかを別に示す |
| 引用・参照 | 分野内での受容・重要度の候補 | 引用のされ方を確認する |
| 出版社紹介 | 内容紹介・目次把握 | 評価根拠としては弱い |
| レファ協/図書館情報 | 調査ルート・存在確認・所蔵確認 | 内容評価とは分ける |
| 個人 Web / ブログ | 発信者確認後、専門家の二次情報になりうる | プロフィール未確認なら探索ヒント |
| 読者レビュー / Amazon レビュー | 探索語・一般的受容の雰囲気 | 学術的評価・内容把握の根拠にしない |
| SNS / 検索スニペット | 最低限の探索ヒント | 本文理解や評価の根拠にしない |

個人 Web、ブログ、note などを根拠として使う場合は、プロフィール、実名または継続的な筆名、所属・肩書、専門分野、researchmap / KAKEN / CiNii / ORCID / 大学ページ等へのリンク、当該テーマとの関連、記事内の出典・引用・ページ番号・書誌情報を確認する。

## 確認優先度

候補をフラットに並べず、調査上の確認優先度を仮に付ける。ただし、出版社・媒体・シリーズだけで文献の価値を確定しない。

確認優先度の手がかり:

- 資料種別: 一次資料、同時代証言、公的資料、研究論文、研究書、新書、評論、書評、パンフレット、個人出版など
- 出版社・媒体: 学術出版社、大学出版会、専門出版社、総合出版社、一般向け新書、個人出版、自費出版、雑誌、学会誌など
- シリーズ: 学術選書、大学出版会叢書、新書、文庫など
- 著者属性: 当該分野の研究者、隣接分野の研究者、ジャーナリスト、当事者、一般著者、不明
- 引用・書評状況: 学術論文で引用されているか、学会誌・専門誌で書評されているか
- 刊行時期: 同時代資料か、後年の研究か、論争後の応答か
- 内容アクセス: 本文確認済みか、要旨のみか、書誌のみか

`優先: 高` / `中` / `低` / `保留` は確認順であり、文献価値の最終判断ではない。自費出版・個人出版支援・オンデマンド出版系、個人出版、通俗的媒体は通常、二次文献・解説的媒体として慎重に扱う。主題一致だけで高優先にしない。著者属性、本文の史料処理、引用・書評状況、対象文献そのものと専門的書評・批判・応答を確認するまでは `優先: 中` または `優先: 保留` に置く。

学会誌・研究会誌・紀要・専門誌の書評、リプライ、書評論文、批判的検討は、一般レビューではなく専門的受容・批判の資料として扱う。中心文献や論争文献を評価するときは、書名・著者名に `書評` / `批判` / `応答` を組み合わせて検索し、見つかったものを候補から落とさない。ただし書評本文を読んだか、書誌だけかは分けて示す。

Web由来情報は補助確認として扱う。出版社・団体の性格、著者属性、本文入口、公開PDF所在、DB外の専門的反応を確認する用途に限り、文献DB由来の書誌・要旨・目次・本文確認とは分ける。Web検索だけで候補集合を作ったり、一般レビューや検索スニペットを学術的評価の根拠にしたりしない。

ただし、著者が当事者である、刊行時期が論争や出来事と近い、本文が同時代証言や受容史を示す、といった条件が確認できる場合は、調査上の価値を個別に評価して残す。

## OCR 固有の注意

- OCR は誤認識を含む。特に以下は誤認識率が高い:
  - 変体仮名・くずし字
  - 旧字体・異体字
  - 縦組み・ルビ付きテキスト
  - 低解像度スキャン
- 重要な主張をする場合は **必ず `page_image_url` で画像目視確認** する
- `coordjson` の座標を使ってトリミング確認するとさらに精度が上がる

## 報告での使い方

```
結果を報告する際は必ず候補確度と本文状態を分ける:

候補確度: 書誌確認済み: 「NDL 書誌IDと CiNii 書誌が一致。本文は未確認」
候補確度: 有力候補: 「OCR ヒットはあるがページ画像未確認」
候補確度: 弱い候補: 「タイトルが類似していますが別資料の可能性があります」
候補確度: 関連性未確定: 「検索結果には出たが、要旨・目次・本文は未確認」

本文読解に基づく断定は `確認: 本文` / `本文: 確認済み` の場合だけ行う。
```
