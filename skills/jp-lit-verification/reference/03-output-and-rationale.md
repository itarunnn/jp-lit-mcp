# jp-lit-verification output and rationale

## 主出力

主出力は表にする。

| 抽出文献 | 推定タイプ | 検証結果 | 判定理由 | 一致した根拠 | 不一致点 | 確認候補 | 次の手 |
|----------|------------|----------|----------|--------------|----------|----------|--------|

## 判定理由

各行では必ず次を説明する。

- 何が一致したか
- 何が一致しなかったか
- なぜその判定にしたか
- 次に何を確認すべきか

## 確認候補

`部分一致` と `混線の疑い` では、確認のために見つかった近い候補を可能な限り併記する。

- 著者名
- 書名
- 掲載誌名または出版社
- 発行年
- source
- 可能なら `source_id` または URL

`highlights` / `summary` / `table_of_contents` がある場合は、確認候補や判定理由に短い抜粋を添える。目的は、元データへ戻らなくても「どこが近いのか」「なぜ怪しいのか」を読み取れるようにすること。

## 会話運用

- 会話へ残すのは表と判定理由が中心
- 詳細書誌や OCR を再確認するのは必要時だけ
- 重要なのは「何が一致し、何が一致しなかったか」であり、検索結果全文の貼り付けではない
