# 問い返しガイダンス（clarifying-questions）

このファイルでは「計画確認」と「問い返し」を区別する。どちらも検索前に行うが、目的が異なる。

---

## 1. 調査計画の確認（全ての依頼で必須）

Skill が動いている場合、全ての依頼で検索前に調査計画を提示してユーザーの確認を取る。
MCP ツールを呼ぶのは確認後のみ。

source の自然言語読み替えは MCP ツール側（jp_lit_search の description）に定義されている。

### 提示フォーマット

使う source・検索語・理由を示す。ユーザーが追加・変更を指示できる余地を作る。

```
以下の source で調べます：
- cinii_articles / ndl_articles（論文）
- ndl_catalog / cinii_books（図書）
検索語: 「〇〇」「△△」（表記ゆれ含む）
よろしければ開始します。追加・変更があればお知らせください。
```

scope が明らかに小さい依頼（単一の書誌確認など）は 1 行で示してもよい:

```
まず ndl_catalog で「○○」を確認します。よいですか？
```

ユーザーが「もっと広く」と言えば source を追加し、「まずざっくりで」と言えば ndl_search（広域・1 リクエスト）から始める方向で計画を更新する。

## 2. 問い返し（intent・資料種別が不明な場合のみ）

### 前提：このスキルが動いている時点で文献調査は確定

「文献調査なのか一般説明なのか」を確認する必要はない。問い返しは intent や資料種別が判断できないときのみ行う。問い返しは最小限（1〜2 問）にとどめる。

### 問い返しが必要なケース

| 状況 | 確認すべきこと |
|------|--------------|
| 資料種別が不明 | 「論文を探していますか、図書（単行本）ですか、一次資料（当時の文書・史料）も含めますか?」 |
| 時代が不明 | 「対象の時代はいつ頃ですか?（近世・近代・現代など）」 |
| 用途が不明 | 「研究・論文執筆用ですか、それとも概要把握が目的ですか?」 |
| 入手可否の優先度が不明 | 「すぐ読める資料（オンライン公開）を優先しますか、所蔵確認だけでよいですか?」 |
| 専門 DB が有効か不明 | 「〇〇の専門 DB（例: nihu_bridge）まで含めますか、一般的な DB で十分ですか?」 |

### 問い返しの書き方

- 一度に複数の質問をしない（1〜2点まで）
- 選択肢を示す（「論文ですか、図書ですか」のように）
- 「わからなければそのまま進めます」と添えて圧をかけない

例:
```
少し確認させてください。
対象は **論文・雑誌記事** ですか、それとも **図書（単行本）や一次資料（史料）** も含めますか?
わからない場合はそのまま進めます。
```

### 問い返し不要のケース

- タイトル・著者・出版年が具体的に分かっている → bibliography_lookup として実行
- 「全文検索して」「デジコレで探して」→ fulltext_page_lookup として実行
- 資料種別・対象が依頼の文言から明らかに読み取れる → 確認なしで計画提示へ進む
