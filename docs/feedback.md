# フィードバックの送り方

jp-lit-mcp は、検索 API を呼ぶ MCP サーバーであると同時に、日本語文献調査の進め方を Skills として配るプロジェクトです。

そのため、通常の bug report だけでなく、調査経験にもとづく「この進め方は違う」「この分野では先に見る DB が違う」「この資料種では断定を避けてほしい」といった feedback が重要です。

## 送ってほしいもの

- 実際に投げた依頼文
- 対象分野・資料種
- 期待した調査の進め方
- 実際の進め方
- 違和感があった判断
- 望ましい検索語、DB、確認順序
- 共有してよい公開資料・根拠

## 送らないでほしいもの

- 未公開の研究テーマや調査計画
- 個人情報
- 権利上共有できない本文・画像・データ
- 利用条件に反する大量転載

## issue template の使い分け

- 不具合、install、schema error、cache/export の問題: `Bug report`
- Skill の調査行動への違和感・改善案: `Skill behavior feedback`
- 新しい DB / source 追加相談: `Source request`

## 反映方針

いただいた feedback は、そのまま「正解」として採用するのではなく、次のどれに当たるかを確認して反映します。

- 汎用ルールとして Skill に入れる
- 特定分野の注意として Skill に入れる
- docs の例や注意書きにする
- source 追加や mapper 改善として扱う
- 利用条件や実装負荷の理由で見送る

文献調査では分野ごとの慣習が大きいため、反映時には断定を増やすより、確認順序・検索語候補・未確認扱いの精度を上げることを優先します。
