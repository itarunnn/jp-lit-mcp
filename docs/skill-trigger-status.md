## 確定した設計方針

### 発火条件

| | 発火条件 | 発火しない例 |
|--|---------|------------|
| **MCP** | `ndl_catalog で` のように source を明示して使う | 「調べて」単体、API ドキュメント調査、Web 検索 |
| **Skill** | `文献DBで` / `文献DBを始めます` を明示。一度発火したらセッション中は継続 | 同上 |

発火時点で文献調査と確定しているため、「文献調査か一般説明か」の確認は不要。検索前には毎回、調査計画の確認を挟む。

### 横断検索の 2 モード

| モード | 呼び方 | 用途 |
|--------|--------|------|
| `jp_lit_search(source=ndl_search)` | **NDL モード** | 存在確認・初動。100 機関以上・1 リクエスト。CiNii/J-STAGE はハーベスト済みで薄い。nihu_bridge 対象外 |
| `jp_lit_search`（source 未指定） | **ラウンドロビンモード** | 詳細調査。ネイティブ API で著者・所蔵・PDF リンクが充実。nihu_bridge 含む |

### quick の原則

- quick を含むすべての深度で対話ファーストを維持する（確認ゼロで即実行する深度は存在しない）
- 深度によって変わるのは確認の**量**だけ：quick = 1 問まで、standard = 1〜2 問、deep = 方針全体を提示して確認
