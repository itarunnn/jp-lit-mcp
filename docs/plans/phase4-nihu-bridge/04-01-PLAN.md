<objective>
`nihu_bridge` を新 SourceName として追加し、`filters.nihu_bridge` の Zod スキーマと SearchParams 型を整備する。これにより以降のプランで adapter 実装と tool 統合を独立に進められる。

Purpose: 型と Zod スキーマは複数のファイル（types / schemas / sources/types / services/searchService）から参照されるため、1 プランで一気に整備して後続を unblock する。
Output: `nihu_bridge` を含むコンパイルが通る状態（実装はまだ無い）。
</objective>

<context>
- 既存パターンは `irdb` をそのまま倣う。`filters.irdb` が `irdbFiltersSchema` で実装されているのと同じ要領で `filters.nihu_bridge` を追加する。
- `SourceName` union（src/lib/types.ts）と `sourceSchema` z.enum（src/lib/schemas.ts）は **両方** 更新する必要がある。順序は既存の他 source の後尾に追加する。
- `searchInputSchema.superRefine` に `filters.nihu_bridge` 用の validation を追加する。`source !== "nihu_bridge"` のとき addIssue する。
- `SearchParams.filters` の型（src/sources/types.ts）にも `nihu_bridge?: NihuBridgeSearchFilters` を追加する。
- `bbox` は数値 4 つ（lat1 / lon1 / lat2 / lon2）のオブジェクト。`institute` は `nijl | nmjh | ninjal | ircjs | rihn | nme | nihu` の z.enum 配列。
</context>

<interfaces>
追加する Zod スキーマと型のシグネチャ:

```typescript
// src/lib/schemas.ts に追加
export const nihuBridgeInstituteSchema = z.enum([
  "nijl", "nmjh", "ninjal", "ircjs", "rihn", "nme", "nihu"
]);

export const nihuBridgeBboxSchema = z.object({
  lat1: z.number(),
  lon1: z.number(),
  lat2: z.number(),
  lon2: z.number()
});

export const nihuBridgeFiltersSchema = z.object({
  institute: z.array(nihuBridgeInstituteSchema).optional(),
  database: z.array(z.string()).optional(),
  normalize: z.boolean().optional(),
  period_from: z.string().optional(),
  period_to: z.string().optional(),
  bbox: nihuBridgeBboxSchema.optional()
});

export type NihuBridgeFilters = z.infer<typeof nihuBridgeFiltersSchema>;
export type NihuBridgeInstitute = z.infer<typeof nihuBridgeInstituteSchema>;
export type NihuBridgeBbox = z.infer<typeof nihuBridgeBboxSchema>;
```

```typescript
// src/sources/types.ts に追加
export interface NihuBridgeSearchFilters {
  institute?: Array<"nijl" | "nmjh" | "ninjal" | "ircjs" | "rihn" | "nme" | "nihu">;
  database?: string[];
  normalize?: boolean;
  period_from?: string;
  period_to?: string;
  bbox?: { lat1: number; lon1: number; lat2: number; lon2: number };
}
```

       ```typescript
       filters: z.object({
         irdb: irdbFiltersSchema.optional(),
         nihu_bridge: nihuBridgeFiltersSchema.optional()
       }).optional()
       ```
       注意: 元の `filters: z.object({ irdb: irdbFiltersSchema }).optional()` は `irdb` が必須の object になっている。これを **両キーとも optional** に変更する（「filters が在るなら irdb と nihu_bridge のどちらも省略可」のセマンティクスにする）。
    3. `searchInputSchema.superRefine` の中の既存 `if (data.filters?.irdb !== undefined && data.source !== "irdb")` の **直後** に同パターンの check を追加する:
       ```typescript
       if (data.filters?.nihu_bridge !== undefined && data.source !== "nihu_bridge") {
         ctx.addIssue({
           code: z.ZodIssueCode.custom,
           message: "filters.nihu_bridge は source=nihu_bridge のときのみ有効です",
           path: ["filters", "nihu_bridge"]
         });
       }
       ```

       重要（source の optional 性）:
       - `searchInputSchema` の `source` フィールドは **optional**（横断検索を許すため省略可能。`source` 省略 = 既定 source 群での横断検索を意味する）。既存 `irdb` の check と同型のため変更不要だが、設計を明示しておく。
       - `data.source` が `undefined` のときは厳密不等価 `!== "nihu_bridge"` が **true** になる（`undefined !== "nihu_bridge"`）。よって「source 省略 + filters.nihu_bridge 指定」のケースも本 check で正しく reject される（プラン 04-04 Task 3 のテスト「source なし + filters.nihu_bridge を reject する」が成立する根拠）。
       - 既存の `irdb` 用 check も同じセマンティクスで動作している（横断検索時に filters.irdb を指定すると reject される）ため、ここで採用する pattern は既存の挙動と一致する。
    4. ファイル末尾の type export 群（`export type IrdbFilters = z.infer<typeof irdbFiltersSchema>;` の付近）に以下を追加する:
       ```typescript
       export type NihuBridgeFilters = z.infer<typeof nihuBridgeFiltersSchema>;
       export type NihuBridgeInstitute = z.infer<typeof nihuBridgeInstituteSchema>;
       export type NihuBridgeBbox = z.infer<typeof nihuBridgeBboxSchema>;
       ```

         database?: string[];
         normalize?: boolean;
         period_from?: string;
         period_to?: string;
         bbox?: { lat1: number; lon1: number; lat2: number; lon2: number };

       ```

         irdb?: IrdbSearchFilters;
         nihu_bridge?: NihuBridgeSearchFilters;
       };
       ```
       注意: 元コードでは `irdb?: IrdbSearchFilters;` がすでに optional なので、行の追加だけ。
